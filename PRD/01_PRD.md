# Claude Agent Office — PRD

## 1. 제품 개요

**한 줄 요약:** 내 Claude Code 에이전트들의 작업 현황을 실시간 모니터링하고, 디스코드에서 소통하며, 원격으로 작업을 지시할 수 있는 AI 에이전트 오피스.

**해결하는 문제:**
- Claude Code 세션이 여러 개 돌아갈 때 일일이 터미널을 확인해야 함
- "허용 대기" 중인 세션을 놓쳐서 작업이 멈춤
- 외출 시 모바일에서 작업 현황 확인/지시 불가
- 보유 스킬/에이전트/플러그인 전체 현황 파악 어려움

**대상 사용자:** 본인 (1인 바이브코더)

**핵심 가치:**
1. 한 화면에서 전체 시스템 파악 (스킬 38개, 에이전트 2개, 플러그인 41개, 세션 ~10개)
2. 디스코드로 어디서든 알림 + 소통
3. 모바일에서 원격으로 새 작업 지시

## 2. 현재 시스템 (As-Is)

| 구성요소 | 상태 |
|----------|------|
| server.js | Node.js HTTP 서버 (포트 3333), 대시보드 API |
| sync.js | 로컬 동기화 에이전트 (세션 수집 → 서버 전송) |
| index.html | 3D 가상 오피스 + 테이블 대시보드 |
| 배포 | 로컬 + Railway 가능 |

## 3. 목표 시스템 (To-Be)

### 3.1 시스템 개요 대시보드
- 스킬/에이전트/플러그인/훅/커맨드 전체 카운트 카드
- 활성 세션 수, 총 토큰 사용량, 변경 파일 수
- CLAUDE.md 기반 규칙 수 표시
- 사이드바 네비게이션: 개요 / 세션 모니터 / 가상 오피스

### 3.2 디스코드 봇
- **알림 기능:**
  - 세션 상태 변경 시 알림 (idle→working, working→waiting)
  - waiting 상태 감지 시 즉시 알림 (허용 필요)
  - 세션 완료/에러 시 알림
- **명령어:**
  - `!status` — 전체 시스템 현황 요약
  - `!sessions` — 활성 세션 목록 + 상태
  - `!detail [프로젝트명]` — 특정 세션 상세 정보
  - `!skills` — 스킬 목록
  - `!trigger "프롬프트" --project=이름` — 원격 작업 실행
  - `!kill [세션ID]` — 세션 중지

### 3.3 원격 트리거
- 디스코드 또는 웹 대시보드에서 새 Claude Code 세션 시작
- 프롬프트 + 대상 프로젝트 디렉토리 지정
- 진행 상태 실시간 전달 (디스코드 채널)
- 완료 시 결과 요약 전달

## 4. 비기능 요구사항

| 항목 | 기준 |
|------|------|
| 응답 시간 | 대시보드 로딩 < 2초 |
| 봇 응답 | 명령어 응답 < 3초 |
| 가동 시간 | 24/7 (Railway 배포) |
| 보안 | 디스코드 채널 권한으로 제어 (로그인 불필요) |
| 동시 세션 | 최대 15개 모니터링 |
