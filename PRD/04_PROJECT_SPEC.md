# Claude Agent Office — 프로젝트 스펙

## 1. 기술 스택

| 레이어 | 기술 | 이유 |
|--------|------|------|
| 서버 | Node.js (기존 server.js 확장) | 이미 구축됨, 언어 전환 불필요 |
| 봇 | discord.js v14 | Node.js 생태계, 성숙한 라이브러리 |
| 프론트엔드 | Vanilla JS + Three.js (기존) | 추가 빌드 불필요 |
| 배포 | Railway | 24/7 봇 가동, $5/월 |
| 데이터 | 메모리 + JSON 파일 | DB 불필요, 간단 |
| 인증 | 없음 (디스코드 채널 권한) | 1인 사용, 간단 |

## 2. 프로젝트 구조

```
claude-dashboard/
├── server.js          # HTTP 서버 + Discord 봇 (통합)
├── sync.js            # 로컬 → 서버 동기화 에이전트
├── bot.js             # Discord 봇 로직 (server.js에서 import)
├── index.html         # 웹 대시보드 (3D 오피스 + 테이블)
├── stats.js           # 시스템 통계 수집 (~/.claude/ 읽기)
├── trigger.js         # 원격 트리거 로직
├── jobs.json          # 원격 작업 기록
├── package.json
├── Procfile           # Railway 배포용
├── .env               # DISCORD_TOKEN, DISCORD_CHANNEL_ID, SYNC_SECRET
└── PRD/               # 기획 문서
```

## 3. 환경변수

| 변수 | 설명 | 필수 |
|------|------|------|
| PORT | 서버 포트 (기본 3333) | 아니오 |
| SYNC_SECRET | sync.js ↔ server.js 인증 | 예 |
| DASH_PASSWORD | 웹 대시보드 비밀번호 | 아니오 |
| DISCORD_TOKEN | 디스코드 봇 토큰 | 예 (Phase 1) |
| DISCORD_CHANNEL_ID | 알림 보낼 채널 ID | 예 (Phase 1) |
| DISCORD_OWNER_ID | 봇 명령 허용할 사용자 ID | 예 (보안) |

## 4. 디스코드 봇 명령어 상세

### Phase 1
| 명령어 | 설명 | 응답 예시 |
|--------|------|----------|
| `!status` | 전체 현황 | "세션 10개 (작업 2, 대기 1) / 스킬 38 / 토큰 2.5M" |
| `!sessions` | 세션 목록 | 임베드: 프로젝트명, 상태, 브랜치, 최근 작업 |

### Phase 2
| 명령어 | 설명 | 응답 예시 |
|--------|------|----------|
| `!trigger "프롬프트"` | 기본 디렉토리에서 실행 | "작업 시작! Job #3" |
| `!trigger "프롬프트" --project=이름` | 특정 프로젝트에서 | "claude-dashboard에서 작업 시작!" |
| `!detail 프로젝트명` | 세션 상세 | Git, 토큰, 최근 액션 등 |
| `!kill 세션ID` | 세션 중지 | "세션 종료됨" |
| `!jobs` | 원격 작업 목록 | 진행/완료/실패 목록 |

### Phase 3
| 명령어 | 설명 |
|--------|------|
| `!skills` | 스킬 목록 (38개) |
| `!pr 프로젝트` | PR 생성 트리거 |
| `!deploy 프로젝트` | 배포 트리거 |

## 5. 알림 규칙

| 이벤트 | 알림 내용 | 우선순위 |
|--------|----------|---------|
| 세션 → waiting | "🟡 [프로젝트] 허용 필요! 터미널 확인" | 높음 |
| 세션 → working | "🟢 [프로젝트] 작업 시작" | 낮음 |
| 세션 완료/에러 | "🔴 [프로젝트] 세션 종료" | 중간 |
| 원격 작업 완료 | "✅ Job #N 완료: [요약]" | 높음 |
| 원격 작업 실패 | "❌ Job #N 실패: [에러]" | 높음 |

## 6. 절대 하지 마 목록

1. **디스코드 봇 토큰을 코드에 하드코딩하지 마** — 반드시 .env 사용
2. **원격 트리거에 인증 없이 접근 허용하지 마** — DISCORD_OWNER_ID 확인 필수
3. **sync.js에서 세션 JSONL 전체를 매번 읽지 마** — 마지막 50줄만 파싱 (기존 방식 유지)
4. **봇을 별도 프로세스로 분리하지 마** — server.js와 같은 프로세스에서 실행 (데이터 공유)
5. **Railway 무료 티어에서 항상 켜두려 하지 마** — 500h/월 제한. 프로 플랜($5) 사용
6. **디스코드에서 rm, 포맷 등 위험한 명령 허용하지 마** — 화이트리스트 방식
7. **Three.js를 무거운 모델로 업그레이드하지 마** — Canvas 기반 경량 유지
